    Helper function to escape a single header parameter using the HTML5 strategy.

    Helper function to escape a single header parameter using the HTML5 strategy. (The alternative would be the strategy defined by RFC5987) Particularly useful for Content-Disposition header parameters which might contain non-ASCII values, like file names. This follows the "WHATWG HTML living standard" section 4.10.21.8 and matches the behavior of curl and modern browsers. See https://html.spec.whatwg.org/multipage/form-control-infrastructure.html#multipart-form-data
